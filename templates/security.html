Security Architecture Overview
SecureVault uses a layered security model. Your master password never leaves your browser in plain text, credentials are encrypted before storage, and the server never holds decryption keys.

Threats Mitigated
  • 🛑
    Brute-Force Attacks
    Argon2id makes GPU/ASIC attacks expensive. zxcvbn blocks weak passwords before they enter the vault.
  • 🛑
    Database Theft
    All stored passwords are Fernet-encrypted. A stolen database file contains only ciphertext — unreadable without the key.
  • 🛑
    Session Hijacking
    Sessions are HMAC-signed by Flask. Tampering with the session cookie invalidates it immediately.
  • 🛑
    Credential Reuse
    The built-in generator creates unique, high-entropy passwords for every site — eliminating the risk of credential stuffing.
